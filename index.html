1.1 Distributed systems
A distributed system is a software system whose components are physically separated and
connected to each other by a network of computers, which communicate and coordinate with each other through
posts. These components interact with each other to achieve a common goal. The systems
distributed allow the availability and ease of access to information, transfer of
data at high speeds, control of access to information transmission, in addition to managing
and integrate information.
The beginnings of this technology began in the 70s, one of the main characteristics of
the distributed systems were the 'mainframes' (super computers of high capacity and efficiency),
but at this time WANDS networks were used, with very slow transmission, and therefore the process

Great strides have also been made in the software's ability to handle large amounts

1.3 SAT problem
NP-complete problems are the most complicated of the NP class, in the sense that if Q 'is
a decision problem in NP and Q is an NP-complete problem, so all instances
of Q 'are polynomially reducible to an instance of Q. The satisfiability problem (SAT)
was the first problem identified as belonging to the NP-complete complexity class by
Stephen Cook in 1971. The problem consists of knowing if, given a Boolean expression with
variables and without quantifiers, there is some assignment of values ​​for your variables that makes the
expression is true. For example, an instance of SAT would be to know if there are values ​​for
x 1 , x 2 , x 3 , x 4 x 1 , x 2 , x 3 , x 4 such that the expression:

be true.
Efficient and scalable algorithms for SAT were developed in the last decade and have con-
contributed to dramatic advances in our ability to automatically resolve instances of
problems involving tens of thousands of variables and millions of constraints (that is, clause
the). On this, a SAT-solver is a tool that takes as input a CNF formula and
generates a successful Boolean assignment to the variables used in the CNF formula if the formula-
mule is satisfactory or not.
The most common classification of algorithms that solve SATs is whether they are complete or incomplete.
[23] .
The full algorithm searches for the solution space of a given SAT problem based on mechanics.
Enumeration and backward search isms. If the problem is satisfactory, then the
solution of the logical formula, and if not, proof that it is complete will be given. In fact, in the second
case, it is extremely important and practical to give proof of integrity, as many cases
Unsatisfactory require proof that they cannot be satisfied. The full algorithm determines the
value of variables in propositional logic one by one until all variables are assigned
and the value of the formula is true, or all possible assignments are tried but remain

complete is an in-depth search process of the entire solution space, the space of

6.1 How do multiple clients make requests to the classification service?
Based on the previously developed architecture, if a client wants to use a service,

ends up executing n threads for n client requests, in distributed services these threads are
isolate (this can be a negative point on shared memory usage) and spread over
the nodes of the network.

6.2 Multiple instances cannot share dependencies
The nature of the network creates a hierarchy of service instances distributed among the
nodes of the same.
This means that when an instance (parent) requests an instance of another service (child or
dependence), the latter will become dependent on the former, the father adopting the right to
decide when the child should stop and the responsibility for his consumption of resources (in case
where a node tracks the consumption of each instance, it must take into account that of
its dependencies). In this way, if the parent is stopped, the node where its child is running
you will be able to stop it.
If this were not the case, a node would not know which instances are left in a 'zombie' state since none of its
clients (parent instances) would take responsibility.
6.3 Json vs GRPC
Json has become practically a standard when it comes to sharing information on the web,
because it is the serialization system of Javascript, and this, in turn, is the standardized language
to run on web browsers.
But in a network of microservices, where the runtime environments are virtualized, Javascript
loses prominence and, therefore, Json as well. This is why it is worth asking which is the best
way to serialize communications between services. As it is evident that there is no protocol
capable of leading neither in any field nor indefinitely, each slot of a service will define the
Tocol supported, this in order to guarantee the scalability of the system architecture.
Specifically, Grpc will be used for this project, mainly for two reasons:
